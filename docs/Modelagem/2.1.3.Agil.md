## 2.1.3.1 Priorização com Moscow
<p align="justify">&emsp;&emsp;O processo de Elicitação de Requisitos foi realizado utilizando duas principais técnicas de elicitação: <a href="https://unbarqdsw2022-1.github.io/2022.1_G2_DonAct/#/Base/1.1.AbordagemNaoEspecifica?id=brainstorm">Brainstorming</a> e <a href="https://unbarqdsw2022-1.github.io/2022.1_G2_DonAct/#/Base/1.1.AbordagemNaoEspecifica?id=introspec%c3%a7%c3%a3o">Introspeção</a>, mas também foram coletados requisitos oriundos de todas as fases do Desing Sprint, resultando assim em 34 Histórias de usuário elicitadas que foram consolidadas no Backlog do produto que visa reunir, organizar e gerenciar os requisitos do sistema. No backlog, as Histórias de usuário foram divididas em 5 Épicos e foram sub-divididos em 15 Features.</p>

<p align="justify">&emsp;&emsp;Nesta etapa de consolidação realizamos a priorização dos Requisitos utilizando a técnica MosCow. O método de priorização MoSCoW é uma técnica amplamente utilizada no mercado definir a prioridade tarefas de um projeto. Criada pelo Engenheiro Dai Clegg nos anos 90, ela foi escolhida pela equipe por ser simples e intuitiva, se adequando bem a projetos com curto prazo.</p>

<p align="justify">&emsp;&emsp;A aplicação da técnica MosCow consiste na classificação das Histórias de usuários em 4 níveis de prioridade: Must have, Should have, Could have e Won't have.</p>

| Id  | Nivél       | Prioridade                    |
| --- | ----------- | ----------------------------- |
| 1   | Must have   | Nessesario que sejá feito     |
| 2   | Should have | Deveria ser feito             |
| 3   | Could have  | Poderia ser feito             |
| 4   | Won't have  | Pouco provável que sejá feito |

<p align="justify">&emsp;&emsp;Após o processo de priorização dos requisitos, como iniciativa extra, foi realizado o desenvolvimento dos Critérios de Aceitação, estes definem um conjunto de condições que devem ser atendidas para que uma História de usuário seja considerada concluída.</p>

function criarTabela(conteudo) {
  var tabela = document.createElement("table");
  var thead = document.createElement("thead");
  var tbody=document.createElement("tbody");
  var thd=function(i){return (i==0)?"th":"td";};
  for (var i=0;i<conteudo.length;i++) {
    var tr = document.createElement("tr");
    for(var o=0;o<conteudo[i].length;o++){
      var t = document.createElement(thd(i));
      var texto=document.createTextNode(conteudo[i][o]);
      t.appendChild(texto);
      tr.appendChild(t);
    }
    (i==0)?thead.appendChild(tr):tbody.appendChild(tr);
  }
  tabela.appendChild(thead);
  tabela.appendChild(tbody);
  return tabela;
}

            if (WIFEXITED(retStatus) != 0){
                gettimeofday(&end, 0);
                printf("> Demorou %.1f segundos, retornou %d\n", timeSpend(fb, end), WEXITSTATUS(retStatus));
            }
        }
    }

    gettimeofday(&end, 0);
    printf(">> O tempo total foi de %.1f segundos\n", timeSpend(begin, end));
}

double timeSpend(struct timeval begin, struct timeval end) {
    long seconds = end.tv_sec - begin.tv_sec;
    long microseconds = end.tv_usec - begin.tv_usec;
    double spend = seconds + microseconds*1e-6;

    return spend;
}
